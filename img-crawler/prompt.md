# 개발 초기

넌 뛰어난 개발자고 앱을 만들려고해
기능 요구사항은 다음과 같아.

```
이미지 크롤러 앱 기능 정의서
1. URL 입력 및 유효성 검사

기능: 이미지 크롤링 대상 웹 페이지의 URL 입력 필드 제공
세부 사항:
입력 URL 유효성 자동 검사 (예: http://, https:// 시작, 유효 도메인 형식)
유효하지 않은 URL 입력 시 친절한 오류 메시지 표시
2. 크롤링 시작 및 진행 상황 표시

기능: 크롤링 시작 버튼 제공
세부 사항:
크롤링 진행 상황 시각적 표시 (예: 로딩 스피너, 진행률 바, 이미지 개수)
크롤링 중단 '중지' 버튼 제공
3. URL 및 이미지 선택자 반복 크롤링

기능: URL 특정 부분 반복 및 여러 이미지 선택자 지정하여 반복 크롤링 지원
세부 사항:
URL 반복:
URL 내 반복할 숫자/문자열 패턴 지정 기능 (예: https://example.com/images/page={})
반복 범위(시작, 끝, 증감 값) 설정 UI 제공
이미지 선택자 반복:
여러 CSS 선택자(selector) 지정 기능 (예: .main-image img, .thumbnail img)
선택자 추가, 삭제, 편집 인터페이스 제공
지정된 선택자 활용 이미지 크롤링 및 중복 제거 다운로드
4. 이미지 추출 및 다운로드

기능: 입력 URL 웹 페이지에서 이미지 식별 및 추출
세부 사항:
<img src="..."> 태그 파싱하여 이미지 URL 추출 (URL 및 이미지 선택자 반복 크롤링 기능과 연동)
추출된 이미지 URL 기반으로 지정된 저장 경로에 다운로드
이미지 파일명 원본 유지 또는 고유 식별자 추가 (예: 원본파일명_타임스탬프.확장자)
중복 이미지 다운로드 방지 로직 (재다운로드 방지)
다운로드 성공/실패 여부 기록 및 표시
5. 저장 경로 설정

기능: 다운로드 이미지 저장 로컬 경로 지정 기능
세부 사항:
기본 저장 경로 제공 및 사용자 변경 가능 (파일 탐색기 연동)
설정된 저장 경로 앱 종료 후 유지
6. 크롤링 결과 요약 및 미리보기

기능: 크롤링 완료 후 다운로드 이미지 개수 및 결과 요약 제공
세부 사항:
크롤링된 이미지 썸네일 또는 목록 미리보기 제공
각 이미지의 원본 URL 및 다운로드된 파일명 표시
오류 발생 이미지 정보 명확히 표시 (예: 다운로드 실패)
7. 에러 핸들링 및 사용자 알림

기능: 크롤링 중 발생 가능한 오류 처리 및 사용자 알림
세부 사항:
각 오류 상황에 대한 명확하고 이해하기 쉬운 메시지 제공
오류 해결을 위한 가이드라인 제시 (예: "네트워크 연결 확인")
```

뭘로 만들지 계획부터 얘기해줘.
